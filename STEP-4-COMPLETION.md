## Step 4.1: Backend Signup Handler

### File Location
```
backend/src/handlers/auth/signup.ts
```

### Implementation Details

**Language:** TypeScript  
**Runtime:** AWS Lambda  
**Framework:** AWS SDK v2

#### Key Features

1. **Input Validation**
2.    - Email format validation
      -    - Password strength validation (8+ chars, uppercase, lowercase, numbers)
           -    - Username validation (alphanumeric, 3-20 chars, unique)
                -    - Display name validation (non-empty, max 50 chars)
                     -    - All validation using Zod schema
                      
                          - 2. **Authentication Flow**
                            3.    - Creates Cognito user with AdminCreateUser API
                                  -    - Sets permanent password using AdminSetUserPassword
                                       -    - Marks email as verified
                                            -    - Associates custom userId with Cognito user attributes
                                             
                                                 - 3. **Data Storage**
                                                   4.    - Creates DynamoDB user profile with following structure:
                                                         -      ```
                                                         -       PK: USER#{userId}
                                                         -        SK: PROFILE
                                                         -         GSI1PK: EMAIL#{email}
                                                         -          GSI2PK: EMAIL#{email}
                                                         -           GSI3PK: USERNAME#{username}
                                                         -            ```
                                                         -           - Initializes user profile with default values
                                                         -          - Supports efficient lookups by email and username
                                                     
                                                         -      4. **Error Handling**
                                                         -     - Handles UsernameExistsException from Cognito
                                                         -    - Handles InvalidPasswordException
                                                              -    - Validates username uniqueness in DynamoDB
                                                                   -    - Validates email uniqueness in DynamoDB
                                                                        -    - Returns appropriate HTTP status codes and error messages
                                                                             -    - Comprehensive console logging for debugging
                                                                              
                                                                                  - #### API Specification
                                                                              
                                                                                  - **Endpoint:** `POST /auth/signup`
                                                                              
                                                                                  - **Request Body:**
                                                                                  - ```json
                                                                                    {
                                                                                      "email": "user@example.com",
                                                                                      "password": "SecurePass123",
                                                                                      "username": "john_doe",
                                                                                      "displayName": "John Doe"
                                                                                    }
                                                                                    ```

                                                                                    **Success Response (201):**
                                                                                    ```json
                                                                                    {
                                                                                      "success": true,
                                                                                      "userId": "uuid-string",
                                                                                      "message": "Account created. Please check your email."
                                                                                    }
                                                                                    ```

                                                                                    **Error Responses:**
                                                                                    - `400`: Validation failed - returns error details
                                                                                    - - `409`: Username or email already exists
                                                                                      - - `500`: Unexpected server error
                                                                                       
                                                                                        - #### Dependencies
                                                                                        - - `aws-sdk`: AWS services (CognitoIdentityServiceProvider, DynamoDB)
                                                                                          - - `uuid`: Generate unique user IDs
                                                                                            - - `zod`: Schema validation
                                                                                             
                                                                                              - #### Environment Variables Required
                                                                                              - - `USER_POOL_ID`: Cognito User Pool ID
                                                                                                - - `DYNAMODB_TABLE`: DynamoDB table name
